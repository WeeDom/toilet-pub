# Story 5.2: Site & Device Management Dashboard

**Epic:** 5 - Central Management Platform
**Status:** ‚ùå TODO

## Description
Create a web-based dashboard for monitoring and managing Guard-e-loo deployments across multiple sites. Provides real-time visibility into device status, events, alerts, and allows remote configuration and control.

## Goals
- Visual site map with device status
- Real-time event feed (motion, alerts, errors)
- Device health monitoring (online/offline, metrics)
- Remote camera control (enable/disable)
- Configuration management
- Historical analytics and reports
- Multi-user with role-based access

## Key Features
- Interactive site map (Leaflet/Google Maps)
- Device status dashboard (cards/grid)
- Real-time event timeline
- Camera live view (when enabled)
- Alert notifications (browser push)
- Analytics charts (occupancy, uptime)
- Device configuration forms
- User management and permissions

## Technology Stack
- Frontend: React or Vue.js
- Maps: Leaflet.js
- Charts: Chart.js or D3.js
- Real-time: WebSocket connection to API
- Styling: Tailwind CSS or Material-UI
- Authentication: JWT tokens

## Acceptance Criteria
- [ ] Dashboard shows all sites on map
- [ ] Device cards show status (online/offline/degraded)
- [ ] Real-time event feed updates without refresh
- [ ] Remote camera enable/disable working
- [ ] Live camera view (when privacy allows)
- [ ] Alert notifications appear in real-time
- [ ] Historical charts (occupancy, uptime)
- [ ] Mobile responsive design
- [ ] Role-based access (admin, operator, viewer)
- [ ] Device configuration editable via UI
